# 一人团队最缺的不是工具，是节奏

独立开发者容易陷入两种极端：要么整天调模型聊天没有交付；要么连续熬夜堆功能，忽视内容、SEO 与债务。下面给一套**按工作日切分**的模板，可按你的时区缩放。

# 周一：选题与约束

- 写清本周唯一主目标（一句话）
- 列出非目标（本周明确不做）
- 更新或确认 Cursor Rules / 关键 Skill 是否仍适用
- 若做内容站：定 1 篇 [洞察专栏](/insights) 题目，避免周末才想起来凑字数

# 周二至周三：垂直实现

- 按 [VibeCoding 五阶段](/insights/vibecoding-workflow-practical) 小步提交
- 每天结束前保证 `build` 通过
- 用 AI Review 做第一轮自检（[审查清单](/insights/ai-code-review-checklist)）

# 周四：重构与测试锚点

- 半天只重构
- 补关键路径测试或手测脚本
- 更新 Changelog 草稿（[发布说明做法](/insights/ai-changelog-release-notes)）

# 周五：发布、内容与复盘

- 合并、部署、冒烟
- 发布或润色本周文章，内链到产品功能页
- 写复盘：做成了什么、卡在哪、下周只改一件事

# 复盘模板（可直接复制）

1. 本周主目标完成度（百分比 + 一句原因）
2. AI 最省时间的三个场景
3. AI 造成返工的三个场景（写进 Rules / Skill 反模式）
4. 新增或更新的 Skill / Rule 列表
5. 技术债条目是否增加（见 [避免技术债](/insights/avoid-ai-tech-debt)）
6. 下周三步以内的计划

# 内容与产品双轮

对 RuleHub 这类站点，**产品功能**与**原创文章**应互相引流：文章解决认知问题，功能承接行动（搜索 Skills、浏览 VibeCoding）。独立开发者若只做聚合不做原创，广告与搜索都不友好；若只写文章不打磨产品，转化弱。

建议比例（可调）：70% 产品，30% 内容；临近 AdSense 或重大传播节点可短期提高内容比重。

# 工具极简集

- 编码：Cursor 或 Claude Code 选一个作主工具
- 资产：团队或个人 Skills 目录 + RuleHub 外部发现
- 信息： [AI 最新消息](/vibecoding/news) 每周浏览一次即可，避免信息过载

# 小结

独立开发的 AI 工作流，核心是把「灵感」关进日历：有主目标、有重构日、有复盘、有内容输出。工具会变，节奏可迁移。

下一步：用本模板规划下周日历，并先写好周一的「非目标」列表——它往往比目标列表更能救你。
