# 目标

结束本路线图时，你应拥有一份针对**真实重复任务**的 `SKILL.md`，并至少用 AI 完整跑通一次。概念预习可先读 [什么是 Agent Skill](/insights/what-is-agent-skill)。

# 0～15 分钟：选题

选本周已经做过两次以上的事，例如：

- 新增一个 UI 组件
- 写一条带校验的 API
- 整理 Changelog

拒绝选题：「让 AI 变得更聪明」这种无法验收的目标。

写下成功标准一句话：「生成的 PR 只需小改即可合并。」

# 15～40 分钟：按结构填空

打开编辑器，按 [格式指南](/insights/skill-md-format-guide) 写：

1. 名称与一句话价值
2. 何时用 / 何时不用
3. 前置条件
4. 分步流程（5～10 步）
5. 完成定义
6. 至少 5 条反模式（[反模式写法](/insights/skill-anti-patterns-guide)）
7. 一个迷你示例

先求可跑，不求完美文笔。

# 40～70 分钟：冷启动试跑

新开 AI 会话，只给 Skill + 任务变量（如组件名），不要额外口述一堆补充——补充应写回文档。

记录：

- 哪一步被跳过
- 哪句被误解
- 你手工改了什么

# 70～85 分钟：修订

把试跑问题改成更硬的句子与反模式。删除空话。确认命令与路径和仓库一致。

# 85～90 分钟：入库

- 放入团队或个人 `skills/` 目录
- 在目录索引登记维护者与日期
- 若可分享，考虑后续整理到开源仓并在 RuleHub 被发现

治理方式见 [团队 Skills 库](/insights/team-skills-library)。

# 验收

- 同事或「未来的你」只凭 Skill 能复现一次任务
- `build` / 测试要求写进完成定义
- 无密钥、无本机绝对路径

# 小结

第一个 Skill 的意义是打通「经验 → 文档 → 再执行」闭环。90 分钟足够产出最小可用版；真正的质量来自之后每次返工的回写。
