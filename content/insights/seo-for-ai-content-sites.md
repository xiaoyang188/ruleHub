# 有页面不等于有收录

给站点加上博客目录，只是起点。搜索引擎更关心：内容是否原创、是否满足查询意图、是否可抓取、是否形成主题集群。对想申请广告的站点，这点尤其关键。

# 专栏应服务清晰主题

RuleHub 的主题是 Agent Skills 与 AI 编程。洞察文章应围绕该主题展开，而不是随意拼接热点关键词。主题集中有助于建立站点专业度。

现有专栏入口：[洞察](/insights)。每篇应用内链连回产品能力（搜索、VibeCoding、文档），让内容与功能互相增强。

# 单页质量要素

- **独立标题与 description**：不要整站共用一句空泛描述
- **可读长度**：教程/指南类建议充实论述，避免两三段凑数
- **结构化小标题**：便于扫描与片段展示
- **发布时间真实**：持续更新比一次性倾倒更可信
- **规范链接**：canonical、sitemap 覆盖文章 URL

# 内链策略

好文章应指向：

- 相关专栏文章（主题簇）
- 可行动的产品页（如 [/search](/search)）
- 必要的法律页（关于、联系）——尤其页脚要稳定可达

避免：整页堆外链、无说明的友链农场、复制外站段落。

# 与聚合内容的边界

Skills 索引、新闻聚合可以是产品价值，但**不能替代原创**。审核视角下，若站点几乎全是抓取摘要、缺少站方观点与教程，风险更高。新闻页应尽量提供站内阅读体验与明确来源，而不是纯跳转壳。

# 运营节奏建议

- 稳定产出：例如每周 1～2 篇深度文
- 更新旧文：工具变更时改 `updatedAt` 与过时段落
- Search Console：提交 sitemap、观察覆盖率与查询

独立开发者可把写作排进周节奏，参见 [一周工作流](/insights/indie-hacker-ai-weekly-workflow)。

# 不要做的事

- 用 AI 批量生成低质伪原创撞库
- 隐藏文字、关键词堆砌
- 买垃圾外链
- 冒充官方或误导下载

这些行为短期可能带来流量幻觉，长期伤害域名。

# 小结

原创专栏要帮到 SEO 与广告审核，关键是主题聚焦、单页质量、内链成网、持续更新，并与聚合功能划清「产品」和「内容资产」的界限。
