# 半吊子多语言比单语言更伤

语言切换入口在、文案未译、或仅部分路由有 locale，会让用户与审核者都感到不专业。若翻译与本地化尚未就绪，**明确以中文为主**并关掉切换，往往更稳妥——RuleHub 即采取了这一策略。

# 技术上仍可保留 locale 前缀

例如统一进入 `/zh/...`，便于日后加 `/en`。但 UI 不要假装已支持十种语言。中间件与链接策略要一致，避免根路径与带语言路径导航表现不同。

# SEO

- `html lang` 与内容语言一致（如 `zh-CN`）
- 元数据与专栏文章用中文标题/描述
- 若暂时无多语言，不必强行上 `hreflang` 空壳

内容质量仍靠原创专栏，见 [SEO 文](/insights/seo-for-ai-content-sites)。

# 文案治理

- UI 字符串集中，避免硬编码散落（即便只有中文）
- 专有词统一：Skill、Rules、VibeCoding
- 新闻聚合若来源英文，站内摘要策略要产品内定（译/不译/部分译）

# 何时再打开语言切换

- 关键页面完成翻译与审校
- 法律页同步多语言或明确适用范围
- 导航、页脚、错误态均已覆盖

# 小结

国际化是产品能力，不是按钮装饰。未就绪时诚实做中文站，路由可预留，切换可关闭。

下一步：检查所有对外入口文案是否均为完整中文，无「EN」无效切换残留。
