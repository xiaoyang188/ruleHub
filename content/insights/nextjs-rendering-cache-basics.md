# 「能打开」不等于「策略对」

AI 常默认生成看起来静态的页面，或反过来把一切标成动态。结果是：专栏不该每请求打文件系统却打了；新闻列表该定时刷新却一直旧数据。

# 粗略选型

| 内容 | 常见策略 |
|------|----------|
| 营销页、法律页、已定稿专栏 | 静态 / SSG |
| 频繁更新的列表 | ISR 或定时再验证 |
| 强个性化、鉴权后数据 | 动态 |

App Router 约定见 [Next.js 结对](/insights/nextjs-app-router-ai-pairing)。

# 与 AI 协作时要说清

「这篇洞察是构建时读 Markdown，不需要请求时读库。」  
「新闻列表最多 5 分钟旧可接受。」

写进页面 Skill 的前置条件，能减少胡乱加 `noStore` 或胡乱缓存。

# 验收

- 改一篇 Markdown 后，生产更新路径是否符合预期（重建或 revalidate）
- sitemap 是否包含新文章
- 动态页是否误伤整站静态优化

# 小结

渲染策略是产品决策。先定新鲜度，再让 AI 写代码，缓存误会会少很多。
